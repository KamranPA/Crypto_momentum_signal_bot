# بک‌تست مرحلهٔ ۱ — استراتژی چندلایه (CoinEx)

## نصب
```bash
pip install -r requirements.txt
```

## اجرا
```bash
python backtest.py
```

نتایج در پوشهٔ `results/` ساخته می‌شود:
- `trades_<SYMBOL>.csv` — جزئیات تک‌تک معاملات هر ارز
- `summary.csv` — خلاصهٔ عملکرد (Win Rate, Profit Factor, Max Drawdown, ...)

## منطق دریافت دیتا
1. اول تلاش می‌شود دیتا از **CoinEx** (از طریق `ccxt`) گرفته شود — چون هدف نهایی معاملهٔ واقعی روی همین صرافی است.
2. اگر تعداد کندل‌های CoinEx برای بک‌تست کافی نبود (طبق حداقل‌های `config.py`)، به‌طور خودکار به **Yahoo Finance** سوییچ می‌شود:
   - روزانه: مستقیم از Yahoo (`interval=1d`)
   - ۴ساعته: چون Yahoo تایم‌فریم 4H ندارد، دیتای ۱ساعته (حداکثر ~۷۳۰ روز گذشته) گرفته و به ۴H تبدیل (resample) می‌شود.
3. این fallback **فقط برای عمق تاریخی بک‌تست** است. اجرای زندهٔ سیگنال‌دهی/معامله باید همیشه از CoinEx بخواند، چون رفتار قیمت/حجم بین صرافی‌ها می‌تواند کمی فرق داشته باشد.

هر بار که اسکریپت اجرا می‌شود، در ترمینال چاپ می‌کند از کدام منبع برای هر نماد استفاده شده — حتماً این خروجی را چک کنید تا بدانید نتایج بر پایهٔ دیتای واقعی CoinEx است یا Yahoo.

## نکات مهم دربارهٔ این نسخه (مرحلهٔ ۱)

### تفاوت با سند اصلی استراتژی — منطق خروج
سند اصلی خروج را **دستی** و بر پایهٔ Swing High/Low تعریف کرده بود. چون این مورد برای بک‌تست خودکار قابل‌کدنویسی مستقیم نیست (تشخیص Swing نیاز به تعریف دقیق پارامتر دارد)، در این نسخه از دو قانون قابل‌اندازه‌گیری استفاده شده:
1. **Stop-loss / Take-profit بر اساس ATR** (`config.ATR_STOP_MULTIPLIER`, `config.ATR_TARGET_MULTIPLIER`)
2. **خروج با برگشت CMF از صفر** (Signal Flip) — همان‌طور که در سند به‌عنوان "حد سود پیشنهادی" آمده بود

اگر می‌خواهید دقیقاً منطق Swing High/Low را هم بک‌تست کنید، باید تابع تشخیص Swing (مثلاً با `scipy.signal.argrelextrema` یا پنجرهٔ N کندلی) اضافه شود — این را می‌توانم در نسخهٔ بعدی اضافه کنم.

### جلوگیری از Look-ahead Bias
- EMA200 روزانه با یک روز تأخیر (`shift(1)`) به جدول ۴ساعته متصل می‌شود، یعنی هر کندل ۴ساعته فقط از EMA200 **روز قبلِ بسته‌شده** استفاده می‌کند، نه روز جاری (که هنوز کامل نشده).
- رژیم BTC هم فقط از کندل ۴ساعتهٔ *بسته‌شدهٔ* هم‌زمان محاسبه می‌شود.

### کارمزد و اسلیپیج
مقادیر فرضی در `config.py` (`FEE_PCT`, `SLIPPAGE_PCT`) هستند — قبل از اعتماد به نتایج، این اعداد را با کارمزد واقعی حساب خودتان در CoinEx (spot یا futures، معمولی یا VIP) چک و اصلاح کنید.

## این چیزی نیست
- این یک **Walk-Forward** نیست — فقط یک بار روی کل بازه با پارامترهای ثابت تست می‌کند. اعتبار واقعی پارامترها (دورهٔ ۲۱، آستانه‌های CMF) فقط با Walk-Forward مشخص می‌شود (مرحلهٔ بعدی).
- این توصیهٔ مالی نیست و نتیجهٔ بک‌تست هیچ تضمینی برای عملکرد آینده نمی‌دهد.

## گام بعدی پیشنهادی
بعد از بررسی `summary.csv`:
- اگر Profit Factor به‌طور معنادار بالای ۱ و تعداد معاملات کافی (حداقل ۳۰-۵۰ در طول بازه) بود → آماده برای Walk-Forward
- اگر ضعیف بود → قبل از رفتن سراغ Walk-Forward، پارامترها یا حتی فرض‌های استراتژی را بازبینی کنید
